# tinyml-explorations

Software-side TinyML experiments — training and exporting models destined for
microcontrollers, de before hardware arrives.

The premise:write deployment-ready inference code on a laptop today,
flash it to a microcontroller the moment hardware is available.

---

## Structure

```
tinyml-explorations/
├── experiments/          # One folder per experiment
│   └── ecg-anomaly-detection/   ← active
├── deployments/          # Hardware-ready C/Arduino code (populated as MCU arrives)
└── notes/                # Technical write-ups and research summaries
```

## Philosophy

Most TinyML tutorials start with hardware. This repository starts without it.
Every experiment produces:
1. A trained model with documented accuracy
2. A C file generated by `m2cgen` that compiles with `arm-none-eabi-gcc`
3. A tradeoff analysis (accuracy vs model size vs inference cost)
